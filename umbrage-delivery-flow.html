Umbrage Delivery Process

End-to-end engagement flow · Product Engineering track
🚀 Engagement Start
↓
Define & Design Figure out what to build — before building anything
Phase 1
Mobilize
Get the team ready. Review the SOW, onboard everyone, set up tools (Miro, Jira, Notion, Figma, Slack), and run the internal kickoff. Align with the client sponsor before Discovery begins.
🏔 SOW Signed
→
Phase 2
Discover
Run the client kickoff meeting. Then investigate — user interviews, stakeholder interviews, surveys, market research, field visits. Share 2–3 visual concepts to align on design direction.
🏔 Client Kickoff
→
Phase 3
Define
Run discovery workshops to validate the problem. Create personas, journey maps, and app flows. Engineering starts ERD and architecture diagrams. Package everything in a Findings Report.
🏔 Findings Report
→
Phase 4
Design
Wireframes → prototypes → hi-fi designs. User testing and iteration. Full product backlog written and estimated in Productboard. Align with client on MVP scope. Package into a Product Guide.
🏔 Product Guide
→
Phase 5
Sprint Prep
Set up Jira, Github, and CI/CD. Load first 2 sprints. Align on team velocity. Run the development kickoff with the client. This marks the end of D&D and the start of building.
🏔 Dev SOW Signed
✅
D&D Exit Gate
Backlog estimated?
First 2 sprints ready?
Dev SOW signed?
Team onboarded?
all four → start building
💡 Intern tip — Define & Design If you join mid-engagement, D&D is likely already done. The key outputs you'll inherit: a Productboard backlog, hi-fi Figma designs, a Findings Report, and a Product Guide. These explain the "why" behind every decision — read them before you do anything else.
↓
Development begins
Develop & Deliver Build in 2-week sprints — the loop repeats until done
Day 1
Sprint Planning
Set the Sprint Goal. Pull stories from the backlog that meet "Definition of Ready." Team commits to what they'll ship this sprint. (1 hr)
→
Every day
Daily Standups
15-min sync. What did you do? What are you doing today? Any blockers? Keeps everyone aligned and problems surfaced fast. (15 min)
→
All sprint
Build & Test
Stories: Drafted → In Dev → In QA → Done. Code: Dev → Staging → UAT → Production. PM stays on top of blockers and keeps the board moving.
→
Mid-sprint
Sprint Grooming
Review upcoming backlog items together. Clarify requirements, estimate stories, resolve open questions. Keeps the next sprint ready before this one ends. (1 hr)
→
End of sprint
Sprint Demo
Show completed work to the client. Gather feedback. Update the backlog based on what you learned. (1 hr)
→
End of sprint
Sprint Retro
Reflect on how the team worked. What went well? What didn't? Commit to 1–2 specific improvements. Then the loop starts again. (30 min)
↩ Loop repeats each sprint until the roadmap is delivered
Running in parallel throughout Develop & Deliver
Continual Discovery

Keep learning from users as we ship — interviews, usability sessions, analytics. Discovery doesn't stop at D&D.

Change Management

Help the client org get ready to adopt what we're building. Training, comms, stakeholder prep.

Program Management

Track risks, assumptions, issues, decisions (RAID). Keep the decision log updated. Spot dependencies early.

Weekly Status Reports

Sent to the client every week. Covers progress, risks, decisions, and what's next. No surprises.

✅
Sprint Exit Gate
Roadmap delivered?
Client accepted?
Ready for handoff?
all three → handoff
💡 Intern tip — Develop & Deliver This is most likely where you'll join. As a PM intern, your job is to keep stories moving: write clear acceptance criteria, facilitate grooming, and make sure engineering is never blocked waiting on a decision or a design.
↓
Handoff begins
Industrialize & Support Hand it off so the client can run it without us
Step 1
Production Launch
Go-to-market activities, readiness checks, feature flags, staged rollout. The product goes live for real users.
→
Step 2
Operational Handoff
Write runbooks, define on-call rotations, set up monitoring (Sentry, Datadog), and document incident response. Umbrage engineers pair with client engineers to transfer ownership.
→
Step 3
Knowledge Transfer
Complete all docs, record architecture walkthroughs, finalize the decision log. The client team should be able to build on top of this without us in the room.
→
Step 4
Learning Loops
Validate outcomes against the goals set in D&D. A/B tests, telemetry, user feedback. Did we actually solve the problem we set out to solve?
✅
Engagement Close
Client self-sufficient?
Docs current?
Outcomes hit?
Client calls us back?
all four → closed
↓
🏁 Engagement Complete
